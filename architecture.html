Agent Breach Replay System Architecture

A privacy-first security trace layer and visual replay studio for tool-using AI agents. The system captures trust, influence, tool boundaries, policy decisions, and violations, then turns them into replayable incidents.

North Star

Product Promise

Show the breach path without collecting the breached data. Store structured security traces, not raw customer emails, files, prompts, or secrets by default.

OpenAI-Facing Positioning

Complement OpenAI tracing and guardrails with a security replay layer that labels sources, detects dangerous influence paths, and explains agent tool failures.

Data Flow

1

Capture

SDK records source reads, tool calls, policy checks, and violations.

2

Redact

Raw sensitive data is stripped, hashed, or shortened before upload.

3

Ingest

API validates trace schema and attaches org, project, and run IDs.

4

Store

Postgres stores durable run, event, policy, and finding records.

5

Detect

Rules identify exfiltration, confused deputy, and unsafe actions.

6

Retrieve

Moss finds similar incidents, prior reports, and guardrail docs.

7

Explain

OpenAI models generate summaries, reports, and recommendations.

Core Data Stores

Source of Truth

Postgres

Stores orgs, projects, agents, runs, trace events, findings, policy decisions, reports, and audit metadata.

Retrieval Layer

Moss

Indexes incident summaries, trace narratives, detector outputs, and guardrail docs for fast semantic search.

Optional Payloads

Object Storage

Stores opt-in exported reports, large redacted artifacts, and attachments that do not belong in relational tables.

Security Predicates

Exfiltration

Protected data reaches an external destination.

Untrusted-to-Action

Untrusted content influences a privileged or real-world action.

Confused Deputy

An attacker causes the agent to misuse its own authority.

Destructive Write

The agent mutates or deletes data through an unsafe chain.

OpenAI Model Usage

Use Models For

  • incident explanations
  • guardrail recommendations
  • security report generation
  • trace normalization assistance
  • policy authoring assistance

Do Not Use Models As

  • the only detector for serious findings
  • a required recipient of raw private data
  • a replacement for deterministic policy checks
  • a hidden authority over allow/block decisions

Deployment Plan

Application

Deploy the Next.js frontend and API routes on Vercel for public launch, previews, environment management, and fast iteration.

Database

Use Neon or Supabase Postgres with connection pooling. Keep trace payloads structured and avoid raw data by default.

SDK Distribution

Publish TypeScript SDK to npm and Python SDK to PyPI. During alpha, support private GitHub installs.
